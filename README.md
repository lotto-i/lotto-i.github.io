# [세계 최초! 로또아이(LOTTOi)](https://lottoi.kr) 결과 공개 검증 백서
*(World's First Public Verification Structure Whitepaper – KR)*

---

> **Since 2025. 02. 08.**  
> **로또아이는 세계 최초로 외부 플랫폼(Google, GitHub 등) 연동형 공개 검증 시스템을 설계하여 운영하고 있습니다.**


## 1. 문서 개요

[로또아이](https://lottoi.kr)는 2025년 2월 8일부터 **세계 최초**로 회원이 생성한 번호를 **추첨 결과가 나오기 이전 시점**에 외부 시스템에 기록해 왔습니다. 

본 시스템은 복권 구매가 마감된 직후(토요일 20:00)부터 추첨 시작 전까지의 모든 데이터를 외부 제3자 시스템(Google)에 저장합니다. 이를 통해 **사후에 당첨 결과에 맞춰 번호를 수정하더라도 그 변경 이력이 대중에게 투명하게 공개**되도록 설계하여, 결과 조작이 원천적으로 불가능한 검증 환경을 제공합니다.

---

## 2. 공개 검증 구조의 기본 원칙

[로또아이](https://lottoi.kr)의 공개 검증 구조는 다음 원칙을 기반으로 합니다.

- **세계 최초 외부 시스템 기록:** 복권 구매 마감 직후(20:00)부터 추첨 전까지 모든 데이터를 외부 시스템에 저장
- **수정 이력 은폐 불가:** 기록 이후 발생하는 모든 **변경 시점과 내용은 Google/GitHub 로그를 통해 기록되어 은폐 불가**
- **검증 주체의 개방성:** 검증의 주체는 운영자가 아닌 **사용자 누구나**

로또아이는 기록의 수정을 단순히 막는 것을 넘어, **수정이 발생하더라도 그 사실을 숨길 수 없게 함으로써 조작의 시도 자체가 무의미해지는 투명한 검증 구조**를 지향합니다.

## 4. GitHub 로그와 시점의 중요성

GitHub 로그와 Google 문서의 이력은 **"데이터의 최종 확정 시점이 추첨 결과 발표 이전인가"**를 입증하는 결정적 지표입니다.

*   **20:00 ~ 20:35 저장:** 추첨 번호가 공개되기 전, 모든 회원 데이터를 외부 저장소에 기록하는 과정입니다.
*   **투명한 이력 관리:** 추첨 이후 누구나 Google 문서의 '버전 기록'을 통해, 데이터가 추첨 시간 이후에 변경되었는지 여부를 직접 확인할 수 있습니다.

GitHub 문서는 필요 시 수정될 수 있으며, 모든 수정은 커밋 이력으로 시간 순서대로 투명하게 공개됩니다.

---

## 5. Google 문서 권한 운영 방식

로또아이는 추첨 전 모든 준비 절차가 외부 시간 기준으로 기록되도록 권한을 엄격히 관리합니다.

*   **5-1. 문서 최초 생성 시:** '읽기 전용(Viewer)' 권한으로 생성하여 사전 임의 수정을 차단하며, 생성 시점은 Google 시스템에 자동 기록됩니다.
*   **5-2. 추첨 당일 (토요일 20:00~20:35):** 회원 생성 번호 저장 직전 '편집' 권한으로 전환됩니다. 이 권한 변경 시점 역시 Google 시스템 로그에 남습니다.

이러한 설계는 운영자의 개입이 발생할 수 있는 모든 순간을 외부 시스템의 기록으로 남겨 검증의 투명성을 확보하기 위함입니다.

---

## 6. 수정 가능성과 변경 이력에 대한 기준

추첨 이후 공개 문서의 편집 권한이 유지될 경우 누군가에 의해 수정이 발생할 가능성은 있습니다. 그러나 로또아이의 시스템에서는 **수정의 발생 여부보다 '수정의 흔적'이 기록된다는 점**이 더 중요합니다.

*   **기록되는 정보:** 변경 시점, 변경 내용, 변경 순서
*   **검증의 원리:** 모든 변경 이력은 Google 시스템에 자동 기록되어 은폐가 불가능합니다. 따라서 누구나 사후 검증을 통해 **'추첨 시간(20:35) 이후에 데이터가 수정되었는지'**를 명확히 가려낼 수 있습니다.

---

## 7. 운영상 수정 발생 시 처리 원칙

URL 오기입 등 운영상의 실수로 인해 추첨 이전에 GitHub 기록이 수정될 수 있습니다. 이 경우에도 로또아이는 다음 정보를 모두 투명하게 공개합니다.

*   최초 기록 시점 및 수정 발생 시점
*   수정 전·후의 URL 내역

검증의 핵심 기준은 **'최종 확정된 URL이 추첨 이전에 존재했는가'**와 **'수정이 추첨 시간(20:35) 이전에 완료되었는가'**입니다.

---

## 8. 외부 공개 검증 시 확인해야 할 핵심 요소

사용자는 다음의 두 가지 경로를 통해 데이터의 무결성을 직접 확인할 수 있습니다.

*   **GitHub 저장소 확인:** Google 문서 URL의 최초 기록 시점이 추첨 이전인지 확인하고, 커밋 이력을 통해 수정 여부 및 시점을 검증합니다.
*   **Google 문서 확인:** 문서 생성 시점과 '버전 기록'을 확인합니다. 특히 **추첨 전 기준 시점(토요일 20:35 이전)**의 최종 변경 내용이 현재의 당첨 결과와 대조했을 때 일치하는지 확인합니다.

## 9. 기술적 한계 및 운영 환경 고지

외부 네트워크 지연, Google/GitHub 플랫폼 장애 등 불가항력적인 요인으로 인해 기록 생성이나 권한 변경 시점이 일시적으로 지연될 수 있습니다.

본 구조의 목적은 지연 없는 완벽한 시스템을 주장하는 것이 아니라, **지연 발생 여부 자체가 은폐되지 않도록 시스템화하는 것**입니다. 실제 생성 시점과 변경 이력은 제3자 시스템에 그대로 남으므로, 사후에 추첨 시점과의 시간 관계를 명확히 판별할 수 있습니다.
